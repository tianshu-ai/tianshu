# SOUL.md — Team agent persona

This file shapes how the orchestrator (天枢) talks on behalf of *your
team*. It is appended to the builtin persona at runtime.

You can leave it empty — the builtin persona will be used as-is.

## Suggested sections

### About the team

> _Who is this team? What kind of work do you do? What's the tone you
> want the agent to adopt — formal/playful, concise/expansive,
> Chinese/English?_

### House rules

> _Conventions every project should respect: file naming, language for
> deliverables, what to never auto-publish, who must approve releases._

### Glossary

> _Internal acronyms or product names the agent should recognise without
> asking._
